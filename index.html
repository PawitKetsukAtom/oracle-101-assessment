🎯 Recommendations

ลำดับการปรับปรุงจากสำคัญไปน้อย — แต่ละขั้น build บนขั้นก่อนหน้า

1️⃣ เสริม Safety hooks ให้ครบ (P0, Effort: S)

ติดตั้ง arra-safety-hooks หรือเขียน pre-push hook เพิ่ม: block rm -rf, block --no-verify, block force push, block push ตรง main ของ project repo — นี่คือจุดที่ ROI สูงสุด (effort เล็ก ผลกระทบใหญ่)

P0 Effort: S Quick Win

2️⃣ สร้าง Heartbeat protocol สำหรับ subagent (P1, Effort: M)

เพิ่ม heartbeat check ทุก 5 นาทีใน OpenClaw subagent — ถ้าเงียบเกิน 10 นาทีให้ capture/escalate — นี่แก้ปัญหา "silent agent" ที่ Oracle 101 เน้นมาก

P1 Effort: M

3️⃣ สร้าง Discord SDLC tag system (P1, Effort: S)

กำหนด tag pipeline: REQ → SRS → DEV → CR → BUG → QA → E2E → DOC → DEPLOY → DONE ใน Discord channels ที่ใช้งานอยู่ — ทำให้ tracking surface เห็นภาพเดียวกัน

P1 Effort: S

4️⃣ ปลุก Faye (QA) ให้ active พร้อม feedback loop (P1, Effort: M)

ตั้งค่า Faye ให้ทำ QA automatically หลัง PR — FAIL ให้กลับ branch/PR เดิม, fail เกิน 3 รอบให้ escalate — ทำให้ pipeline 13 steps เริ่มสมบูรณ์

P1 Effort: M

5️⃣ สร้าง CMMI docs template สำหรับ trading system (P1, Effort: M)

สร้าง SRS/SDD/UAT/RTM template สำหรับ StochasticReversal project — ใช้ markdown, เพิ่ม doc-update mandate ใน PR flow — นี่คือจุดเริ่มต้นของ CMMI Level 3

P1 Effort: M

6️⃣ ศึกษาและพิจารณา maw-js (P0, Effort: L)

maw-js เป็น investment ระยะยาว — ให้เริ่มจากการศึกษา Core 12 commands และทดลองใช้กับ fleet 2 ตัว (Anya + Alice) ก่อน — อย่า migrate ทั้งหมดในครั้งเดียว

P0 Effort: L Long-term

7️⃣ เพิ่ม event triggers (P1, Effort: M)

สร้าง trigger สำหรับ agent-idle, agent-crash, qa-failed, pr-merged — ใช้ OpenClaw cron หรือ webhook + Lambda ก็ได้ — ทำให้ระบบดูแลตัวเองได้มากขึ้น

P1 Effort: M

8️⃣ สร้าง troubleshooting decision tree (P2, Effort: S)

เขียน 5-layer decision tree ใน TOOLS.md หรือ skill แยก — แยกชั้น Agent/Skill → Oracle Memory → HTTP API → Orchestration → UI ตาม Oracle 101

P2 Effort: S

9️⃣ กำหนด worktree policy ตาม project (P2, Effort: S)

แบ่ง repo เป็น Strict (trading system), Preferred (general software), Unrestricted (tools/docs) — และตั้ง git hook ตาม level

P2 Effort: S

🔟 T3 Federation เมื่อพร้อม (P2, Effort: L)

เริ่มจาก maw-js ก่อน (ข้อ 6) — เมื่อคุ้นกับ maw แล้วค่อยเปิด T3 Federation สำหรับงานยาวที่ต้อง survive session death

P2 Effort: L Future
